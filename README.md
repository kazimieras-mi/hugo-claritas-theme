## Features

* Strict, geometric proportions
* Responsive design
* Light / dark mode (system preference)
* Alternating row colors for tables
* Syntax highlighting
* Styled `<mark>` elements
* RSS feed support
* `{{< posts >}}` shortcode
* 404 page with post suggestions
* Print stylesheet

## Installation

### Grab a Copy

Download the contents of this repository, and add them to your <Hugo Project>/themes/claritas.

Set `theme = "claritas"` in your `hugo.toml`. See `exampleSite/` for a working configuration.

### Git Submodule

Git submodule "mounts" the main branch of this theme to your git repository. You may publish these repositories as all license files are included here. I do not use submodules in my repository, which means there won't be nested submodules, which can be a hassle sometimes.

```bash
git submodule add https://github.com/kazimieras-mi/hugo-claritas-theme themes/claritas
```

Set `theme = "claritas"` in your `hugo.toml`. See `exampleSite/` for a working configuration.

## Content Structure

Posts go in `content/posts/`. The `{{< posts >}}` shortcode renders a dated list of all posts and can be placed anywhere, e.g. on the home page.

If your content structure is different, you may need to edit layouts/partials/posts.html.

## Syntax Highlighting

The included `syntax.css` was generated by Hugo's built-in [Chroma](https://github.com/alecthomas/chroma) highlighter. It is not original to this theme. To use it, enable class-based highlighting in your `hugo.toml`:

```toml
\\\[markup]
  \\\[markup.highlight]
    noClasses = false
```

Without this, Hugo defaults to inline styles and the file has no effect. To regenerate or swap the stylesheet with a different Chroma style:

```bash
hugo gen chromastyles --style=monokai > assets/syntax.css
```

## Image Classes

|Class|Effect|
|-|-|
|`small-image`|Small image|
|`tiny-image`|Tiny image|
|`inverting`|Inverts colors in dark mode|

Generally, you'll need to use the HTML <img> tag for this, and that will be possible if you enable raw HTML in Hugo.

## Headless Table Trick

Empty header cells are hidden entirely, enabling headerless tables:

```markdown
||||
|------|------|------|
|Cell A|Cell B|Cell C|
```
